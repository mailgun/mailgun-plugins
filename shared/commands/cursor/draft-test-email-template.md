# Draft a Test Email Template

Ask for purpose, audience, sender domain, template name, subject, personalization variables, and whether the user wants HTML, text, or both.

Draft the content first. Recommend Validate for recipients and Inspect preview QA for HTML before sending or publishing.

Before any tool call that creates a template version, creates an Inspect preview test, or sends an email, confirm the exact domain, template name/version, sender, recipient, subject, and whether test mode should be used.
